# 候选池覆盖率分析报告

## 1. 概述

当前候选池（V4_POOL_SIZE=26）的平均覆盖率为 **70.8%**（3.54/5），低于目标覆盖率（如77.5%）。

## 2. 覆盖率分布

| 覆盖程度 | 期数 | 占比 |
|---------|------|------|
| 5/5 | 15 | 13.5% |
| 4/5 | 44 | 39.6% |
| 3/5 | 39 | 35.1% |
| 2/5 | 12 | 10.8% |
| 1/5 | 1 | 0.9% |
| 0/5 | 0 | 0.0% |

**分析**：大多数期数覆盖3-4个号码，但只有13.5%的期数能覆盖全部5个号码。

## 4. 主要问题分析

### 4.1 偏移评分权重过高

**问题**：当前偏移评分逻辑：
- 偏移0（重号）：15分
- 偏移1（邻号）：10分
- 偏移2：6分
- 偏移3：3分
- 偏移4-5：1分
- 偏移>5：0分

**影响**：号码25、34、35的平均偏移距离较大（4.3-7.0），导致评分过低。

**示例**：
- 2026040 → 2026041：号码25偏移4，仅得1分
- 2025132 → 2025133：号码25偏移6，得0分

### 4.2 热号惩罚过重

**问题**：热号评分中，`hot === 0` 时减2分。

**影响**：对于近期未出现的冷号，即使历史频率正常，也会被过度惩罚。

**数据**：排除率高的号码平均热度都很低（0.1-0.8），说明它们近期出现频率不高。

### 4.3 区间竞争激烈

**问题**：
- 区间1（1-12）：12个号码
- 区间2（13-24）：12个号码  
- 区间3（25-35）：11个号码
- 池大小：26个
- 区间保底：每个区间至少3个

**影响**：区间3有11个号码，但池中只有约8-9个位置（26/3≈8.7），竞争激烈。

### 4.4 历史频率依赖过重

**问题**：依赖近50期的历史频率，但某些号码本身出现频率就低。

**影响**：历史频率低的号码即使有机会出现，也会因评分低而被排除。

## 5. 优化建议

### 5.1 调整偏移评分权重

**当前**：
```javascript
if (minOffset === 0) score += 15;
else if (minOffset === 1) score += 10;
else if (minOffset === 2) score += 6;
else if (minOffset === 3) score += 3;
else if (minOffset <= 5) score += 1;
```

**建议**：
```javascript
if (minOffset === 0) score += 15;
else if (minOffset === 1) score += 10;
else if (minOffset === 2) score += 7;
else if (minOffset === 3) score += 5;
else if (minOffset === 4) score += 3;
else if (minOffset === 5) score += 2;
else if (minOffset <= 8) score += 1;
```

**预期效果**：提高中等距离号码的评分，增加其进入候选池的机会。

### 5.2 减少热号惩罚

**当前**：
```javascript
if (hot >= 4) score += 10;
else if (hot >= 3) score += 7;
else if (hot >= 2) score += 4;
else if (hot === 0) score -= 2;
```

**建议**：
```javascript
if (hot >= 4) score += 10;
else if (hot >= 3) score += 7;
else if (hot >= 2) score += 4;
else if (hot === 0) score -= 1; // 减少惩罚
```

**预期效果**：给冷号更多机会，特别是那些历史频率正常但近期未出现的号码。

### 5.3 增加池大小

**当前**：V4_POOL_SIZE = 26

**建议**：增加到 28-30

**预期效果**：
- 增加2-4个位置，可以容纳更多中等评分的号码
- 提高覆盖率，特别是对于竞争激烈的区间3

### 5.4 优化区间保底策略

**当前**：minIv = [3, 3, 3]（每个区间至少3个）

**建议**：minIv = [3, 3, 2] 或 [3, 2, 3]

**原因**：区间3（25-35）只有11个号码，要求至少3个可能限制了灵活性。

### 5.5 增加冷号保护

**新增逻辑**：
```javascript
// 冷号保护：近5期未出现但历史频率正常的号码
if (hot === 0) {
  const historicalFreq = calculateHistoricalFreq(n, sourceRow);
  const avgFreq = calculateAverageFreq(sourceRow);
  if (historicalFreq >= avgFreq * 0.8) {
    score += 2; // 给予补偿
  }
}
```

**预期效果**：保护那些有潜力但近期未出现的号码。

## 6. 预期改进效果

通过以上优化，预计：
1. **覆盖率提升**：从70.8%提升到75-78%
2. **排除率降低**：高排除率号码的排除率降低10-15个百分点
3. **平衡性改善**：各区间号码分布更均衡

## 7. 实施建议

### 7.1 优先级排序

1. **高优先级**：调整偏移评分权重（直接影响大）
2. **中优先级**：增加池大小（简单有效）
3. **中优先级**：减少热号惩罚（风险较低）
4. **低优先级**：优化区间保底（需要更多测试）
5. **低优先级**：增加冷号保护（新增逻辑）

### 7.2 测试方案

1. **A/B测试**：在script回测.js中分别测试优化前后的覆盖率
2. **逐步实施**：每次只实施一个优化，观察效果
3. **回撤机制**：如果优化导致覆盖率下降，立即回撤

## 8. 结论

候选池覆盖率低的主要原因是：
1. **偏移评分权重过高**，导致中等距离号码被过度排除
2. **热号惩罚过重**，对冷号不公平
3. **池大小不足**，无法容纳所有有潜力的号码
4. **区间保底策略僵化**，限制了灵活性

通过调整评分权重、增加池大小、优化区间策略，可以显著提高候选池覆盖率，从而提升整体命中率。
