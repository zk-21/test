## 三、问题分析

### 3.1 优化效果有限
- 平均尾号命中率仅提升0.3%
- Top5组合命中率仅提升0.8%
- 尾号6和7仍然被预测0次
- 尾号2、8、9仍然被严重低估

### 3.2 根本原因
1. **多样性保障机制位置不当**：在所有信号计算完成后才添加，此时某些尾号已经获得很高分数
2. **信号源重叠问题未解决**：多个信号源仍然给相似尾号重复加分
3. **globalFreq信号权重不足**：15的权重无法平衡其他信号的偏差

### 3.3 典型案例分析
```
优化后案例：
预测: [4, 5, 1, 3, 9]  ← 仍然偏向特定尾号
实际: [2, 7, 8, 0, 4]
漏掉: [2, 7, 8, 0]     ← 漏掉的尾号与预测尾号相邻
```

## 四、进一步优化建议

### 4.1 激进权重调整
```javascript
const weights = {
  sameOrNeighbor: 5,       // 大幅降低：从15→5
  globalFreq: 25,          // 大幅提升：从15→25
  overlap1: 2,             // 降低：避免重叠
  arith1: 3,               // 降低
  crossRowArithmetic: 5,   // 大幅降低：从12→5
  intraGroupPattern: 5,    // 大幅降低：从12→5
  tailArithmeticExtension: 5, // 大幅降低：从12→5
};
```

### 4.2 强制多样性机制
在函数开头添加强制机制，确保每个尾号都有基础分数：
```javascript
// 强制基础分数（确保每个尾号都有机会）
const BASE_SCORE = 50;
for (let t = 0; t <= 9; t++) {
  scores.set(t, BASE_SCORE);
}
```

### 4.3 动态权重调整
根据历史偏差动态调整权重：
```javascript
// 动态调整：如果某个尾号历史预测偏差过大，则提升其权重
const historicalBias = calculateHistoricalBias(sourceRow, allBalls);
historicalBias.forEach((bias, tail) => {
  if (bias < -20) { // 严重低估
    scores.set(tail, scores.get(tail) + Math.abs(bias) * 0.5);
  }
});
```

## 六、结论

当前优化效果有限，主要原因是：
1. 多样性保障机制位置不当
2. 信号源重叠问题未解决
3. 权重调整不够激进

需要采取更激进的优化策略，包括：
1. 大幅降低sameOrNeighbor权重
2. 大幅提升globalFreq权重
3. 在函数开头添加强制基础分数
4. 实施动态权重调整机制
