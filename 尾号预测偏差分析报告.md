# 尾号预测偏差分析报告

基于163期回测数据，分析尾号预测系统的偏差问题。

## 根本原因分析

### 1. 信号源偏差
多个信号源可能过度依赖历史数据中的奇数尾号模式：
- `sameOrNeighbor`（权重15）：基于前一期尾号，可能继承偏差
- `crossRowTail`（权重10）：跨期尾号重复，可能强化偏差
- `intraGroupPattern`（权重12）：组内尾号模式，可能学习偏差模式

### 2. 权重不平衡
虽然`globalFreq`信号已启用（权重15），但可能不够高，无法抵消其他信号的偏差。

### 3. 多样性机制不足
虽然有奇偶平衡机制，但：
- 只确保Top5中至少2个偶数尾号
- 没有确保所有偶数尾号都有合理预测机会
- 尾号6、7、8的基础分数可能太低

### 4. 频率权重调整不足
当前的频率权重调整（`FREQUENCY_WEIGHT = 0.10`）可能过于温和，无法有效纠正偏差。

## 改进建议

### 1. 调整权重配置
```javascript
const weights = {
  // ... 其他权重
  globalFreq: 25,  // 提高全局频率权重
  sameOrNeighbor: 10,  // 进一步降低
  // ... 其他权重
};
```

### 2. 增强多样性机制
- 确保所有偶数尾号都有合理的预测机会
- 添加针对特定尾号（如6、7、8）的补偿机制
- 考虑添加"偶数尾号保底分数"

### 3. 调整频率权重
```javascript
const FREQUENCY_WEIGHT = 0.20;  // 从10%提高到20%
```

### 4. 添加尾号补偿机制
针对预测频率过低的尾号（如6、7、8），添加专门的补偿逻辑。

### 5. 信号源校准
检查并校准可能导致偏差的信号源，确保它们不会过度放大历史偏差。

## 结论
当前预测系统存在严重的尾号预测偏差，过度预测奇数尾号，严重低估偶数尾号。这种偏差导致系统漏掉了大量实际出现的尾号（尤其是2、6、7、8、9）。需要从权重调整、多样性增强、频率校准等多个方面进行系统性优化。
