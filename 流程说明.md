# 选号系统完整流程说明

## 概述
这是一个基于历史数据分析的彩票选号系统，主要针对前区（1-35）选5个号码。系统采用V5版本，结合了多种统计分析和预测算法。

## 流程概览
```
数据收集 → 候选号码生成 → 组合生成 → 组合评分 → 多样性选择 → 最终输出
```

## 详细流程

### 1. 数据收集阶段
**函数**: `collectBalls()`
- 收集所有历史开奖数据
- 数据结构：每个球包含 `row`（期号）、`zone`（前区/后区）、`number`（号码）、`color`（颜色）

### 2. 候选号码生成阶段
**函数**: `buildSampleNumbersV4()` 或 `buildSampleNumbers()`

#### 2.1 数据准备
- **源行选择**: 获取用户选择的期号（`sourceRow`）
- **源号码**: 获取源行的前区号码（`selectedNumbers`）
- **源尾号**: 提取源号码的尾号（0-9）

#### 2.2 预测模型计算
1. **桥接映射** (`buildV4BridgeMap`)
   - 分析号码间的桥接关系（间隔、端点）
   - 为每个号码计算桥接分数

2. **等差映射** (`buildV4ArithmeticMap`)
   - 分析号码间的等差关系
   - 计算等差端点分数

3. **+10期趋势** (`buildV4PlusTenTrendMap`)
   - 分析号码在+10期出现的趋势
   - 计算目标号码的趋势分数

4. **参考行** (`buildV4FullReferenceRows`)
   - 获取历史参考行数据
   - 用于后续的模式匹配

#### 2.3 尾号预测
1. **尾号转移分析** (`analyzeTailTransitionsV4`)
   - 分析尾号之间的转移概率
   - 基于50-70期历史数据

2. **增强尾号预测** (`predictLikelyTailsV4Enhanced`)
   - 融合转移概率、参考行关系、全局高频
   - 输出预测尾号列表（按概率排序）

#### 2.4 首位球预测
**函数**: `predictFirstBallComprehensive()`
- 融合5种预测规律：
  1. ±3范围
  2. ±1相邻
  3. +9期规律
  4. 等差延伸
  5. 尾号转移

#### 2.5 区间比预测
**函数**: `predictTargetIntervalRatio()`
- 基于历史转移概率和时效加权
- 预测下一期的区间分布（一区/二区/三区）

#### 2.6 候选号码评分
对每个候选号码（1-35）进行多维度评分：

1. **偏移评分**: 与源号码的距离关系
2. **尾号关联**: 与预测尾号的匹配度
3. **+10期趋势**: 趋势分数
4. **桥接分数**: 桥接映射分数
5. **等差分数**: 等差映射分数
6. **热号分数**: 近期出现频率
7. **极端期加成**: 极端情况下的调整
8. **连号附近奖励**: 与连号的关系
9. **区间平衡**: 区间分布平衡
10. **奇偶平衡**: 奇偶比例平衡
11. **和值贡献**: 对和值的贡献
12. **历史频率**: 历史出现频率
13. **首位球预测**: 首位球预测匹配

#### 2.7 候选池生成
- 按评分排序，取前13-20个号码作为候选池
- 确保候选池覆盖所有预测尾号

### 3. 组合生成阶段
**函数**: `buildSampleFrontCombosV5()`

#### 3.1 维度独立评分
将候选号码按8个维度独立评分：

1. **尾号维度**: 预测尾号匹配
2. **偏移维度**: 与源号码的距离
3. **热号维度**: 近期出现频率
4. **频率维度**: 历史频率+邻号
5. **桥接维度**: 桥接分数
6. **等差维度**: 等差分数
7. **重复维度**: 与源号码重叠
8. **跨期维度**: 跨期关系

#### 3.2 归一化处理
- 将每个维度的分数映射到0-100
- 确保各维度权重可比

#### 3.3 加权随机抽取
- 从每个维度独立生成组合
- 使用加权随机抽取（`weightPower=0.5`）
- 每个维度生成1-3组组合

#### 3.4 组合基础过滤
对生成的组合进行基础过滤：
1. **跨度过滤**: 14-32
2. **奇偶过滤**: 不能全奇或全偶
3. **区间过滤**: 单区间不超过4个
4. **连续过滤**: 最长连续不超过3
5. **密集过滤**: 间隔≤2的不超过2对

#### 3.5 完整评分体系（14个信号维度）
对通过过滤的组合进行详细评分：

1. **尾号匹配** (`tailMatches`): 与上期尾号匹配度
2. **尾号等差** (`tailArithmetic`): 组合内尾号等差关系
3. **尾号支持** (`tailSupport`): 尾号历史支持度
4. **桥接间隔命中** (`bridgeGapHits`): 桥接间隔命中数
5. **桥接端点命中** (`bridgeEndpointHits`): 桥接端点命中数
6. **等差端点命中** (`arithmeticEndpointHits`): 等差端点命中数
7. **与选中行重叠** (`selectedOverlap`): 与源号码重叠数
8. **与选中行相邻** (`selectedNeighborHits`): 与源号码相邻数
9. **连续对** (`consecutivePairs`): 连续对数量
10. **最长连续** (`longestRun`): 最长连续长度
11. **连续段** (`consecutiveSegments`): 连续段数量
12. **与上期重叠** (`lastRowOverlap`): 与上期号码重叠
13. **与上期相邻** (`lastRowNeighborHits`): 与上期号码相邻
14. **重复支持** (`repeatSupport`): 重复号码支持度
15. **跨期等差** (`hasCrossAP`): 跨期等差关系（存在性检测）

**评分权重**:
- 使用 `sampleSignalLevel` 函数（限制在0-3范围）
- 使用 `sampleRuleWeight`（固定值8）
- 尾号匹配权重更高（`sampleRuleWeight + 2`）

### 4. 组合评分阶段
**函数**: `v5RescoreCombos()`

对组合进行二次评分，考虑更多约束条件：

#### 4.1 和值约束
- **全局和值范围**: 20-170
- **区间比-和值约束**: 不同区间比对应不同和值范围
- **动态调整**: 基于区间方向预测调整和值范围

#### 4.2 区间比约束
- **常见区间比奖励**: 2:1:2、2:2:1等常见比例加分
- **预测区间比匹配**: 与预测区间比匹配度
- **区间方向预测**: 基于历史变化趋势

#### 4.3 锚点变换评分
- **锚点保持**: 与源号码重叠的数量
- **锚点偏移**: 与源号码的距离关系
- **锚点覆盖率**: 源号码被覆盖的程度

#### 4.4 重复号奖励
- 1个重复号：+8
- 2个重复号：+3
- 0个重复号：-5
- 3个及以上：-12

#### 4.5 区间方向预测调整
- **和值方向调整**: 基于区间变化趋势
- **区间稳定时**: 增加重号和尾重权重
- **区间分布匹配**: 与预测方向匹配

#### 4.6 连号评罚
- 4连：-15
- 3连：-5
- 2连：+5
- 1连：+3

#### 4.7 跨度和奇偶
- **跨度**: 18-24最优（+18），26-33次优（+12）
- **奇偶**: 1奇4偶（+12），3奇2偶（+8）

#### 4.8 区间均衡
- 三区都有号码：+5
- 单区间≥3个：惩罚

#### 4.9 尾号模式
- **尾号多样性**: 5个不同尾号+4，4个不同尾号+2
- **尾号连续**: 双连+5，三连+10，四连+15，五连+20
- **尾号等差**: 等差3个+8，等差4个+12

#### 4.10 跨期尾号关系
- **跨期连续**: 相同尾号+2，相邻尾号+1
- **跨期等差**: 步长2-4的等差关系
- **跨行尾号重复**: 与前2-3期的尾号重复

#### 4.11 首位球预测
- **预测匹配**: 与首位球预测的匹配度
- **静态规则**: 95.8%首位球≤15

### 5. 多样性选择阶段
**函数**: `v5GenCombosRandom()` 中的多样性选择

#### 5.1 排序
- 按最终评分降序排序

#### 5.2 多样性过滤
- **重叠阈值**: 重叠≥4的组合跳过
- **去重**: 避免重复组合
- **数量限制**: 最终选择8-20组组合

### 6. 最终输出阶段
**函数**: `sampleButton` 事件处理器

#### 6.1 组合计划生成
**函数**: `buildV4SingleSamplePlan()`
- 生成最终的组合计划
- 包含前区和后区组合

#### 6.2 后区组合
**函数**: `buildSampleFreeCombos()`
- 后区（1-12）选2个号码
- 桥接组合优先

#### 6.3 显示和保存
- 在界面上显示生成的组合
- 保存到历史记录
- 支持导出Excel

## 参数配置

### 关键参数
- `sampleRuleWeight`: 8（规则权重）
- `sampleWeakRuleWeight`: 4（弱规则权重）
- `samplePickCount`: 5（前区选号数量）
- `sampleBackPickCount`: 2（后区选号数量）
- `sampleComboPoolSize`: 13-20（候选池大小）

### 优化参数
- `weightPower`: 0.5（加权随机指数）
- `maxPool`: 13-20（最大候选池）
- 重叠阈值: 4（多样性过滤）
