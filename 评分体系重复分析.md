# 组合生成阶段评分体系重复分析

## 问题

用户疑问：组合生成阶段维度独立评分和完整评分是否合理，存不存在重复的问题？

## 当前评分体系

### 1. 6维度独立评分（用于生成组合）

```javascript
// 8个维度独立评分
dims = [
  { arr: dimTail, count: 3, maxPool: 10 },   // 维度1：尾号
  { arr: dimOff,  count: 1, maxPool: 10 },   // 维度2：偏移
  { arr: dimHot,  count: 1, maxPool: 10 },   // 维度3：热号
  { arr: dimFreq, count: 2, maxPool: 10 },   // 维度4：频率+邻号
  { arr: dimBr,   count: 1, maxPool: 10 },   // 维度5：桥接
  { arr: dimAr,   count: 2, maxPool: 10 },   // 维度6：锚点/等差
  { arr: dimRepeat, count: 1, maxPool: 10 }, // 维度7：重复号码
  { arr: dimCross,  count: 1, maxPool: 10 }, // 维度8：跨期关系
];
```

**作用**：每个维度独立生成组合，不跨维度融合求和

## 重复问题分析

### 存在重复的评分项

| 维度独立评分 | 完整评分 | 重复内容 |
|--------------|----------|----------|
| 维度1：尾号 | 预测尾号匹配 | 都基于预测尾号评分 |
| 维度7：重复号码 | S9: 重复号奖励 | 都对源号码重复加分 |
| 维度8：跨期关系 | 跨期尾号关系 | 都对跨期尾号关系加分 |
| 维度6：锚点/等差 | S7: 锚点变换评分 | 都对锚点变换加分 |

### 重复的具体分析

#### 1. 尾号相关重复

**维度独立评分（维度1）**：
```javascript
// 维度1：尾号
if (topPT.has(t)) s += 35;           // 预测尾号命中
else if (predictTails.some(...)) s += 15;  // 邻号
else if (srcTails.indexOf(t) >= 0) s += 8; // 源尾号
```

**完整评分（预测尾号匹配）**：
```javascript
// 预测尾号匹配
var topTails = new Set(predictTails.slice(0, 5).map(p => p[0]));
var tailMatches = comboTailsArr.filter(t => topTails.has(t)).length;
bonus += tailMatches * 3;  // 每个匹配+3分
```

**问题**：尾号预测在两个阶段都被评分，可能导致过度偏向尾号匹配的组合

#### 2. 重复号码相关重复

**维度独立评分（维度7）**：
```javascript
// 维度7：重复号码
if (srcSet.has(n)) s += 30;          // 源号码本身
else if (srcTails.indexOf(n % 10) >= 0) s += 8; // 源尾号
```

**完整评分（S9: 重复号奖励）**：
```javascript
// S9: 重复号奖励
var repeatCnt = nums.filter(n => srcSet.has(n)).length;
if (repeatCnt === 1) bonus += 8;
else if (repeatCnt === 2) bonus += 3;
else if (repeatCnt === 0) bonus -= 5;
else if (repeatCnt >= 3) bonus -= 12;
```

**问题**：重复号码在两个阶段都被评分，可能导致过度偏向重复号的组合

#### 3. 跨期关系相关重复

**维度独立评分（维度8）**：
```javascript
// 维度8：跨期关系
if (srcSet.has(n)) s += 12;
if (src.some(a => Math.abs(a - n) === 1)) s += 18;
if (srcTails.indexOf(n % 10) >= 0) s += 10;
```

**完整评分（跨期尾号关系）**：
```javascript
// 跨期尾号关系
comboTailsArr.forEach(t => {
  if (prevTails.includes(t)) crossConsec += 2; // 相同尾号
  else if (prevTails.some(pt => Math.abs(pt - t) === 1)) crossConsec += 1;

if (crossConsec >= 6) bonus += 8;
else if (crossConsec >= 4) bonus += 5;
else if (crossConsec >= 2) bonus += 2;
```

**问题**：跨期关系在两个阶段都被评分，可能导致过度偏向跨期关系的组合

## 设计合理性分析

### 合理之处

1. **两个阶段目标不同**：
   - 维度独立评分：生成多样化的候选组合
   - 完整评分：对组合进行质量评估和排序

2. **维度独立评分的作用**：
   - 确保每个维度都能产生组合
   - 避免某个维度被过度压制
   - 提供多样化的候选池

3. **完整评分的作用**：
   - 综合考虑所有约束
   - 对组合进行最终排序
   - 确保组合质量

### 不合理之处

1. **重复评分导致过度偏向**：
   - 尾号匹配可能被过度强化
   - 重复号码可能被过度强化
   - 跨期关系可能被过度强化

2. **权重叠加问题**：
   - 同一个评分项在两个阶段都被加分
   - 可能导致某些组合被不合理地高估

3. **缺乏协调机制**：
   - 两个阶段的评分没有明确的权重分配
   - 可能导致评分结果不稳定

## 改进建议

### 方案1：去除重复评分项

**维度独立评分**：专注于号码选择
- 保留：尾号、偏移、热号、频率、桥接、等差
- 移除：重复号码、跨期关系（留给完整评分）

**完整评分**：专注于组合质量
- 保留：和值、区间比、跨度、奇偶、连号
- 保留：尾号模式、重复号、跨期关系

### 方案2：调整权重分配

**维度独立评分**：降低重复项权重
- 尾号：35 → 20
- 重复号码：30 → 15
- 跨期关系：12+18+10 → 8+12+6

**完整评分**：保持或微调
- 保持现有权重

### 方案3：分阶段评分

**阶段1：号码筛选**（维度独立评分）
- 只考虑号码级别的特征
- 生成候选号码池

**阶段2：组合生成**（维度独立评分）
- 从候选池中生成组合
- 不考虑组合级别特征

**阶段3：组合评估**（完整评分）
- 考虑组合级别特征
- 对组合进行最终排序

## 总结

| 方面 | 当前状态 | 问题 | 建议 |
|------|----------|------|------|
| 尾号 | 两阶段都评分 | 重复 | 降低维度独立评分权重 |
| 重复号码 | 两阶段都评分 | 重复 | 降低维度独立评分权重 |
| 跨期关系 | 两阶段都评分 | 重复 | 降低维度独立评分权重 |
| 锚点变换 | 两阶段都评分 | 重复 | 降低维度独立评分权重 |
| 和值/区间比 | 只在完整评分 | 合理 | 保持 |
| 跨度/奇偶 | 只在完整评分 | 合理 | 保持 |
| 连号 | 只在完整评分 | 合理 | 保持 |

**核心问题**：尾号、重复号码、跨期关系在两个阶段都被评分，可能导致过度偏向这些特征的组合。

**建议**：降低维度独立评分中重复项的权重，或将其移至完整评分阶段统一处理。
