# 当前生成示例逻辑详细说明

## 1. 入口点与参数初始化

**位置**：`sampleButton` 点击事件（行6684-6883）

**初始参数**：
- `requestedRow`：用户指定的起始行号（1-rows）
- `displayMode`：显示模式（"single" 或 "variants"）
- `version`：算法版本（"v3" 或 "v4"）
- `ratioPlan`：区间比计划（从 `getActiveSampleRatios()` 获取）
- `sourceStartRow`：数据源行号
- `frontRepeatTarget`：前区重复目标（0-3随机）
- `backRepeatTarget`：后区重复目标（0-2随机）

## 2. 两种主要模式分支

### 2.1 尾号模式（当 `tailPool.length >= 5` 时）

**触发条件**：用户选择了尾号，且候选池号码数 ≥ 5

**生成流程**：

#### 步骤1：生成后区组合
```javascript
const backSample = buildSampleNumbers(requestedRow, "back");
const backCombos = buildSampleFreeCombos(backSample.candidateEntries, sampleBackPickCount, backRepeatTarget, backSample.selectedNumbers);
```
- 使用原始后区生成逻辑
- 候选池基于历史数据（蓝球区域）
- 评分系统与前区一致

#### 步骤2：生成前区组合
```javascript
const tailEntries = tailPool.map(number => ({ number, score: 0 }));
if (ratioPlan.length > 0 && ratioPlan[0].length === 3) {
  tailCombos = buildSampleFrontCombos(tailEntries, ratioPlan, samplePickCount, sampleIntervals, ...);
} else {
  tailCombos = buildSampleFrontCombos(tailEntries, [], samplePickCount, sampleIntervals, ...);
}
```
- **候选池**：尾号筛选后的号码池（1-35中尾号符合的号码）
- **评分系统**：完全复用 `buildSampleFrontCombos` 的复杂评分逻辑
- **区间比支持**：如果选择了区间比，严格按区间比生成组合

#### 步骤3：选择组合（按分数排序）
```javascript
const comboIndex = i % tailCombos.length;
const selectedCombo = tailCombos[comboIndex];
combo = selectedCombo.numbers.slice(0, 5).sort((a, b) => a - b);
```
- **不再随机选择**：按 `compareSampleCombos` 排序后的顺序选择
- **循环使用**：`i % tailCombos.length` 增加多样性
- **回退机制**：如果没有生成组合，回退到随机选择

#### 步骤4：补漏6优化
```javascript
const top5Covered = new Set(); // 5组组合已覆盖的号码
const top5Freq = new Map(); // 号码出现频率
const c6Scored = tailPool.filter(n => !top5Covered.has(n) || (top5Freq.get(n) || 0) >= 1)
  .map(n => {
    let s6 = 0;
    if (!top5Covered.has(n)) s6 += 30; // 未覆盖加分
    if (freq >= 3) s6 += 25; // 频率加分
    // 邻近加分...
    return { number: n, score6: s6 };
  });
```
- **目标**：选出5个号码，补充未覆盖的高频号码
- **评分维度**：
  - 未覆盖加分（+30）
  - 频率加分（+10/+15/+25）
  - 邻近加分（+3/+6/+12）
- **结果**：生成"补漏6"行

#### 步骤5：显示候选池
- 显示完整的尾号候选池（所有符合尾号的1-35号码）
- 用于用户参考和验证

### 2.2 普通模式（未选尾号或尾号池 < 5）

**触发条件**：未选择尾号，或尾号候选池号码数 < 5

**生成流程**：

#### 步骤1：生成前区候选池
```javascript
const frontSample = version === "v4"
  ? buildSampleNumbersV4(sourceStartRow, "front", ratioPlan)
  : buildSampleNumbers(sourceStartRow, "front", ratioPlan);
```

**v3版本**（`buildSampleNumbers`，行3127-3226+）：
- **数据源**：历史红球数据（1-35区域）
- **评分维度**：
  1. **重复次数**（`repeatCount`）：号码在参考行中出现的次数
  2. **尾号命中**（`tailCount`）：尾号在历史数据中的出现频率
  3. **参考行匹配**：
     - 选定行命中（`selectedRowHits`）
     - 支持行命中（`supportHits`）
     - 最后行命中（`lastRowHits`）
  4. **尾号模式**（`tailPatternScore`）：尾号转移模式匹配
  5. **桥梁端点**（`bridgeEndpointHits`）：桥梁结构端点命中
  6. **等距端点**（`arithmeticEndpointHits`）：等距数列端点命中
  7. **上色区域**（`upperColorHits`）：绿球/黑球区域命中
- **输出**：`candidateEntries`（带分数的候选号码）、`selectedNumbers`（源行号码）、`ratioSupportMap`（区间比支持度）、`referenceRows`（参考行数据）

**v4版本**（`buildSampleNumbersV4`，行4542-4739）：
- **增强评分维度**（在v3基础上增加）：
  1. **尾号转移预测**（`predictedTails`）：基于历史尾号转移概率预测
  2. **区间比预测**（`ivPrediction`）：预测下期区间比分布
  3. **奇偶平衡**（`targetOdd`）：预测下期奇偶数比例
  4. **和值贡献**（`targetSum`）：预测下期和值范围
  5. **+10期趋势**（`plusTenTrend`）：近10期号码出现趋势
  6. **桥梁结构**（`bridgeMap`）：号码间的桥梁关系
  7. **等距结构**（`arithMap`）：等距数列关系
  8. **热号分析**（`hotness`）：近10期高频号码
  9. **极端期检测**（`extremeFlags`）：窄跨度、和值崩溃、奇偶翻转
  10. **历史频率**（`historyMetrics`）：历史出现频率、近期频率、重复率
  11. **区间平衡奖励**：根据预测区间比补充弱势区间
  12. **奇偶平衡奖励**：根据预测奇偶比调整
  13. **和值贡献奖励**：根据预测和值选择大小号码
- **候选池优化**：
  - 保底机制：每个区间至少3个号码
  - 池大小：`V4_POOL_SIZE`（默认20个）
  - 区间平衡：确保三个区间都有代表
- **输出**：除了v3的输出外，还包含`predictedTails`、`ivPrediction`、`candidates`（完整候选池）

#### 步骤2：生成后区候选池
```javascript
const backSample = buildSampleNumbers(sourceStartRow, "back");
```
- **数据源**：历史蓝球数据（1-12区域）
- **评分逻辑**：与v3前区逻辑相同，但独立运行
- **特殊处理**：后区不受区间比约束（1-12为单一区间）

#### 步骤3：生成组合
```javascript
const frontCombos = isV4
  ? buildSampleFrontCombosV4(...)
  : buildSampleFrontCombos(...);
const backCombos = buildSampleFreeCombos(...);
```

**v3前区组合**（`buildSampleFrontCombos`，行2815-2874）：
- **输入**：`candidateEntries`、`ratioPlan`、`samplePickCount`（5）、`sampleIntervals`
- **逻辑**：
  1. 如果有区间比：按区间比生成组合
     - 将候选号码按区间分桶
     - 从每个区间选取指定数量的号码
     - 组合所有可能性
  2. 如果无区间比：生成自由组合
     - 从所有候选号码中选取5个
  3. 评分：使用`buildSampleComboRecord`计算每个组合的分数
  4. 排序：按`compareSampleCombos`排序
- **输出**：带评分的组合数组，按分数降序排列

**v4前区组合**（`buildSampleFrontCombosV4`，行4765）：
- **增强评分**：更多评分维度（S4/S7/S8/S9/S10/S5-S6）
- **多样性选择**：确保组合间的差异性
- **预测尾号整合**：优先选择匹配预测尾号的组合

**后区组合**（`buildSampleFreeCombos`，行2876-2892）：
- **简化逻辑**：无区间比约束
- **评分**：使用`buildSampleComboRecord`的简化版
- **输出**：带评分的后区组合数组

#### 步骤4：创建显示计划
```javascript

  ? buildV4SingleSamplePlan(...)

    ? buildSampleVariantPlan(...)
```

**v3单组模式**（`buildSingleSamplePlan`，行3074-3100）：
- **策略**：从评分最高的组合中循环选择5行
- **前区选择**：`getSampleDisplayPool`获取显示池（考虑`rotationCursor`）
- **后区选择**：类似逻辑
- **输出**：5行相同的"第1行"到"第5行"

**v3变体模式**（`buildSampleVariantPlan`，行3036-3072）：
- **5种策略**：
  1. **最优**（"best"）：选择评分最高的组合
  2. **次优**（"next"）：选择评分次高的组合
  3. **备选**（"backup"）：选择评分第三的组合
  4. **轮换**（"rotate"）：在最高分组合中轮换选择
  5. **随机**（"random"）：从前5名中随机选择
- **选择逻辑**（`pickSampleComboByStrategy`）：
  - 使用`getSampleComboPreferenceOrder`确定偏好顺序
  - 避免重复选择（`usedKeys`）
  - 考虑`rotationCursor`增加多样性
- **输出**：5行不同的策略组合

**v4单组模式**（`buildV4SingleSamplePlan`，行4889-5014）：
- **覆盖优先策略**：`selectCoverageOptimalCombos`选择5个组合
  - Top1：选择最高分组合
  - Top2-5：贪心选择覆盖最多未覆盖候选池号码的组合
  - 覆盖数相同时偏好高分组合
- **补漏6**：额外生成1行补漏组合
  - **评分维度**：
    1. 候选号码基础分数
    2. 尾号匹配（预测尾号）
    3. 区间平衡（补充弱势区间）
    4. Top5频率（出现次数）
    5. 邻近加分（与已选号码的距离）
  - **结构优化**：生成多个候选组合，选择结构最优的
    - 和值范围：65-115
    - 跨度范围：12-28
    - 奇偶比：1-4个奇数
    - 区间覆盖：三个区间都有代表
- **输出**：6行（5行主组合 + 1行补漏6）

## 3. 评分系统详解

### 3.1 组合评分（`buildSampleComboRecord`）

**主要评分维度**：

#### 基础分数
- **重复号码惩罚**：`repeatPenalty`（负分）
- **连号惩罚**：`runPenalty`（负分）
- **跨度惩罚**：`spreadPenalty`（负分）

#### 模式匹配加分
- **尾号匹配**：`selectedTailOverlapScore`（+12到-12）
- **参考行匹配**：`tailReferenceScore`、`arithmeticScore`、`bridgeScore`
- **区间比支持**：`ratioSupport`（+分）

#### 硬过滤惩罚（V3.1新增）
- **全奇/全偶**：-600分
- **跨度极端**：<3或>34：-800分
- **和值极端**：<25或>160：-800分
- **超三连**：>3连号：-800分
- **单区间≥5球**：-800分

### 3.2 组合排序（`compareSampleCombos`）

**排序优先级**（从高到低）：
1. 总分（score）
2. 可解释数量（explainableCount）
3. 覆盖加分（explainCoverageBonus）
4. 变换数量（transformedCount）
5. 变换多样性加分（transformDiversityBonus）
6. 远偏移数量（farOffsetCount）
7. 锚点保持惩罚（anchorKeepPenalty，越小越好）
8. 锚点覆盖数量（anchorCoverageCount）
9. 参考行满足数量（referenceSatisfiedRows）
10. 跨度惩罚（spreadPenalty，越小越好）
11. 最大窗口计数（maxWindowCount，越小越好）
12. 跨度（span，越大越好）
13. 重复惩罚（repeatPenalty，越小越好）
14. 连号段数量（runSegmentCount，越小越好）
15. 最长连号（longestRun，越小越好）
16. 区间比支持（ratioSupport，越大越好）

## 4. 区间比支持

### 4.1 所有可能的区间比（21种）
前区1-35分为3个区间：
- 区间1：1-12
- 区间2：13-24
- 区间3：25-35

所有组合：a-b-c，其中a+b+c=5
- 0-0-5, 0-1-4, 0-2-3, 0-3-2, 0-4-1, 0-5-0
- 1-0-4, 1-1-3, 1-2-2, 1-3-1, 1-4-0
- 2-0-3, 2-1-2, 2-2-1, 2-3-0
- 3-0-2, 3-1-1, 3-2-0
- 4-0-1, 4-1-0
- 5-0-0

### 4.2 断区选项
- break1：区间1为0（0-1-4, 0-2-3, 0-3-2, 0-4-1）
- break2：区间2为0（1-0-4, 2-0-3, 3-0-2, 4-0-1）
- break3：区间3为0（1-4-0, 2-3-0, 3-2-0, 4-1-0）

## 6. 关键函数说明

### 6.1 `getTailFilteredPool()`（行54-61）
- 功能：根据选择的尾号生成候选池
- 逻辑：遍历1-35，尾号符合的号码加入池
- 返回：号码数组

### 6.2 `buildSampleFrontCombos()`（行2815-2874）
- 功能：生成前区组合并评分
- 参数：
  - `entries`：候选号码数组
  - `ratios`：区间比计划
  - `pickCount`：选取数量（默认5）
  - `intervals`：区间划分
- 逻辑：
  1. 如果有区间比，按区间比生成组合
  2. 否则生成自由组合
  3. 使用 `buildSampleComboRecord` 评分
  4. 按 `compareSampleCombos` 排序

### 6.3 `buildSampleNumbers()`（行3127-3226+）
- 功能：生成候选号码池
- 参数：
  - `selectedRow`：数据源行
  - `zone`：区域（"front"或"back"）
  - `ratios`：区间比
- 逻辑：
  1. 收集历史数据中的球
  2. 计算各种评分指标
  3. 生成候选号码及其分数
  4. 返回 `candidateEntries` 和 `selectedNumbers`

### 6.4 `buildSampleFreeCombos()`（行2876-2892）
- 功能：生成自由组合（无区间比约束）
- 用于：后区组合生成
- 逻辑：简单组合生成 + 评分

### 6.5 `buildSampleComboRecord()`（行2423-2522+）
- 功能：计算单个组合的详细评分
- 返回：包含所有评分维度的对象
- 主要指标：
  - `score`：总分
  - `runPenalty`：连号惩罚
  - `spreadPenalty`：跨度惩罚
  - `repeatPenalty`：重复惩罚
  - `ratioSupport`：区间比支持度
  - 各种参考行匹配分数

## 7. 当前逻辑总结

### 优势
1. **完全复用评分系统**：尾号模式使用与普通模式相同的复杂评分逻辑
2. **严格区间比**：选择区间比时，严格按区间比生成组合
3. **按分数排序**：不再随机选择，按评分排序选择组合
4. **补漏优化**：补漏6机制确保覆盖更多候选号码
5. **颜色统一**：前后区使用相同颜色方案
6. **后区独立**：后区不受尾号限制，使用原始生成逻辑

### 改进点
1. **尾号模式的后区**：虽然使用原始逻辑，但候选池可能较小
2. **补漏6的评分**：当前是简单评分，可考虑更复杂的覆盖算法
3. **变体模式**：尾号模式下暂不支持变体模式（只有单组模式）

## 8. 使用建议

1. **选择尾号**：如果希望基于特定尾号生成，选择1-5个尾号
2. **选择区间比**：可选择21种区间比之一，或使用断区选项
3. **查看候选池**：尾号模式下会显示完整的候选池
4. **理解补漏6**：补漏6行是为了补充未覆盖的高频号码
5. **颜色识别**：紫色为示例号码，黑色叠加表示重复出现
