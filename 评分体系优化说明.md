# 评分体系优化说明

## 修改内容

### 1. 移除维度独立评分中的重复项

**修改前（8个维度）**：
```javascript
var dims = [
  { arr: toArr(nTail), count: 3, maxPool: 10 },   // 维度1：尾号
  { arr: toArr(nOff),  count: 1, maxPool: 10 },   // 维度2：偏移
  { arr: toArr(nHot),  count: 1, maxPool: 10 },   // 维度3：热号
  { arr: toArr(nFreq), count: 2, maxPool: 10 },   // 维度4：频率+邻号
  { arr: toArr(nBr),   count: 1, maxPool: 10 },   // 维度5：桥接
  { arr: toArr(nAr),   count: 2, maxPool: 10 },   // 维度6：锚点/等差
  { arr: toArr(nRepeat), count: 1, maxPool: 10 }, // 维度7：重复号码 (移除)
  { arr: toArr(nCross),  count: 1, maxPool: 10 }, // 维度8：跨期关系 (移除)
];
```

**修改后（6个维度）**：
```javascript
var dims = [
  { arr: toArr(nTail), count: 4, maxPool: 10 },   // 尾号：3→4 (提升权重)
  { arr: toArr(nOff),  count: 2, maxPool: 10 },   // 偏移：1→2 (提升权重)
  { arr: toArr(nHot),  count: 2, maxPool: 10 },   // 热号：1→2 (提升权重)
  { arr: toArr(nFreq), count: 3, maxPool: 10 },   // 频率+邻号：2→3 (提升权重)
  { arr: toArr(nBr),   count: 2, maxPool: 10 },   // 桥接：1→2 (提升权重)
  { arr: toArr(nAr),   count: 3, maxPool: 10 },   // 锚点/等差：2→3 (提升权重)
];
```

### 3. 保持完整评分阶段不变

完整评分阶段（v5RescoreCombos）中的重复相关评分项保持不变：

**重复号奖励（S9）**：
```javascript
if (repeatCnt === 1) bonus += 8;
else if (repeatCnt === 2) bonus += 3;
else if (repeatCnt === 0) bonus -= 5;
else if (repeatCnt >= 3) bonus -= 12;
```

**跨期尾号关系**：
```javascript
// 跨期连续关系
if (crossConsec >= 6) bonus += 8;
else if (crossConsec >= 4) bonus += 5;
else if (crossConsec >= 2) bonus += 2;

// 跨期等差关系
if (crossAP >= 4) bonus += 6;
else if (crossAP >= 2) bonus += 3;

// 跨行尾号重复
if (repeatCount >= 3) bonus += (lookback === 2 ? 5 : 3);
else if (repeatCount >= 2) bonus += (lookback === 2 ? 3 : 2);
```

**预测尾号匹配**：
```javascript
bonus += tailMatches * 3;
```

## 优化效果

### 1. 消除重复评分问题

**修改前**：
- 尾号匹配：维度独立评分(+35分) + 完整评分(+15分) = **+50分**
- 重复号码：维度独立评分(+30分) + 完整评分(+8分) = **+38分**
- 跨期关系：维度独立评分(+40分) + 完整评分(+10分) = **+50分**

**修改后**：
- 尾号匹配：完整评分(+15分) = **+15分**
- 重复号码：完整评分(+8分) = **+8分**
- 跨期关系：完整评分(+10分) = **+10分**

### 2. 保持组合生成能力

**修改前**：8个维度，总count=12
**修改后**：6个维度，总count=16

总count从12增加到16，确保能生成足够的组合（至少8注）。

### 3. 提升其他维度权重
