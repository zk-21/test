# 跨行尾号+组内规则改进总结

## 改进日期
2026年7月3日

## 改进目标
通过"跨行尾号"和"组内尾号规则"改进`predictLikelyTailsV4Enhanced()`尾号预测函数，提升候选池覆盖率和Top5命中率。

## 基线结果（改进前）
- **Top5最高命中率**: 31.8%
- **Top5+补漏6 联合覆盖率**: 61.4%
- **候选池覆盖率 (Top30)**: 87.9%

## 改进内容

### 1. 新增跨行尾号分析函数 `analyzeCrossRowTails()`
**位置**: `script回测.js` 第6521行

**逻辑**:
- 统计前2-3期的尾号在历史中的重复频率
- 给前2期尾号+2分、前3期尾号+1分（跨行加权）
- 基于70期历史数据标准化分数到0-10分

**原理**: 跨行尾号重复在历史数据中有显著规律，与前2-3期相同的尾号在下一期出现概率更高。

### 2. 新增组内尾号规则分析函数 `analyzeIntraGroupTailPatterns()`
**位置**: `script回测.js` 第6595行

**逻辑**:
- **尾号多样性**: 分析一组开奖号码中不同尾号的个数（4-5个不同尾号的模式更常见），权重4
- **连续尾号**: 分析尾号连续模式（如1,2,3或8,9,0），权重3
- **等差尾号**: 分析尾号等差模式（如1,3,5或2,5,8），权重3
- 基于70期历史数据标准化分数

**原理**: 历史开奖号码的尾号组合有多样性偏好，了解这些模式可以提升预测准确性。

### 3. 修改 `predictLikelyTailsV4Enhanced()` 函数
**位置**: `script回测.js` 第6157行

**新增权重**:
```javascript
crossRowTail: 15,        // 跨行尾号：与前2-3期尾号重复
intraGroupPattern: 12,   // 组内尾号规则：历史尾号组合模式
```

**新增分析步骤**:
- 第10步：跨行尾号规则分析
- 第11步：组内尾号规则分析

## 改进策略说明
1. **预测尾号**: 通过改进的`predictLikelyTailsV4Enhanced()`函数预测Top5尾号
2. **加入邻号**: 预测尾号 ±1 的邻号也加入候选集
3. **所有匹配尾号入池**: 1-35中所有尾号匹配的号码入池
4. **区间保底**: 确保三个区间都有号码
5. **分数补齐**: 按评分排序补齐到25个候选号码

2. 改进效果需要在完整179期数据上验证
