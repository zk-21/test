### 2.2 关键发现

1. **系统性盲区**：
   - 尾号 **6** 被预测 **0次**，但实际出现 **65次**（40.1%）
   - 尾号 **4** 仅预测12次，但实际出现79次（48.8%）
   - 尾号 **8** 仅预测10次，但实际出现71次（43.8%）

2. **过度预测**：
   - 尾号 **1** 预测158次（97.5%），但实际只出现77次（47.5%）
   - 尾号 **3** 预测152次（93.8%），但实际只出现84次（51.9%）

3. **预测模式固化**：
   - Top5预测几乎总是 **[1, 3, 5, 7, 9]** 或其子集
   - 这些都是**奇数尾号**，系统严重偏向奇数

| 0 | 50次 | 30.9% | 10.5% | 34.6% |

### 3.2 漏掉尾号与预测尾号的关系

**关键发现**：漏掉的尾号与预测尾号相邻的比例高达 **91.8%**

这意味着：
- 系统预测了尾号 **X**
- 实际出现了尾号 **X±1**（相邻尾号）
- 但系统没有预测到这些相邻尾号

**示例**：
- 预测 [5, 3, 7, 1, 9]
- 实际 [6, 8, 2, 4, 5]
- 漏掉 [6, 8, 2, 4] 全部是预测尾号的相邻数字

## 四、根本原因分析

### 4.1 权重配置失衡

```javascript
const weights = {
  sameOrNeighbor: 35,    // 相同或相邻尾号（85.6%概率）
  crossRowArithmetic: 24, // 跨行等差
  globalFreq: 20,        // 已禁用！
  tailArithmeticExtension: 18,
  intraGroupPattern: 18,
  // ... 其他权重较低
};
```

**问题**：
1. `sameOrNeighbor` 权重过高（35），导致预测过度依赖前一期尾号
2. `globalFreq` 被禁用，失去了全局频率统计的平衡作用
3. 多个权重指向相似的尾号（相同/相邻），造成分数累积

### 4.2 信号源重叠

多个信号源都在给相似的尾号加分：

| 信号源 | 目标尾号 | 权重 |
|--------|----------|------|
| sameOrNeighbor | 前一期尾号±1 | 35 |
| overlap1 | 前一期尾号 | 6 |
| arith1 | 前一期尾号±2 | 8 |
| bridgeTails | 前一期尾号之间的中间值 | 12 |

**结果**：如果前一期有尾号5，那么尾号4、5、6会从多个信号源获得高分，而尾号0、2、8等远离的尾号被忽略。

### 4.3 历史数据利用不足

1. **只看短期**：主要依赖前一期（sourceRow-1），忽略长期统计规律
2. **禁用信号**：`globalFreq`、`partialMatch`、`highFreqMiss` 都被禁用
3. **样本偏差**：70期回测窗口可能包含特定模式，导致过拟合

### 4.4 尾号独立性假设错误

系统假设"前一期出现的尾号更可能再次出现"，但数据显示：
- 每个尾号的实际出现频率相对均衡（32%-52%）
- 前一期出现的尾号在下一期出现的概率约为50%，并非显著更高
- **过度依赖前一期信息会导致系统性偏差**

## 六、优化建议

### 6.1 立即可做的调整

1. **降低 `sameOrNeighbor` 权重**：从35降到15-20
2. **启用 `globalFreq`**：添加全局频率统计，平衡预测偏差
3. **增加尾号多样性**：确保Top5预测覆盖奇数和偶数尾号

### 6.2 权重优化方案

```javascript
const weights = {
  sameOrNeighbor: 15,      // 降低：避免过度依赖前一期
  globalFreq: 20,          // 启用：平衡全局频率
  tailArithmeticExtension: 18,
  intraGroupPattern: 12,
  crossRowArithmetic: 10,
  bridgeTails: 8,
  // ... 其他权重保持或微调
};
```

### 6.3 策略性调整

1. **强制覆盖偶数尾号**：在Top5预测中至少包含2个偶数尾号
2. **相邻尾号扩展**：当预测尾号X时，考虑将X±1也加入候选
3. **多期融合**：不仅看前一期，还要看前3-5期的尾号分布

## 七、结论

**命中率不高的根本原因是**：

1. **预测偏差严重**：系统几乎只预测奇数尾号（1,3,5,7,9），完全忽略偶数尾号（0,2,4,6,8）
2. **权重配置失衡**：`sameOrNeighbor`权重过高，导致预测过度依赖前一期尾号
3. **信号源重叠**：多个信号源给相似尾号加分，造成分数累积而非分散
4. **禁用关键信号**：`globalFreq`等平衡信号被禁用，失去纠偏能力

**预期改进**：
- 调整权重后，尾号命中率有望从50.4%提升到60-65%
- Top5组合命中率有望从32.7%提升到40%+
